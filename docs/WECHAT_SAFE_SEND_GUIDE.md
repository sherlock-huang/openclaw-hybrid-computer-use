# 微信安全发送指南

> **核心原则**：任何一步验证失败，消息**绝不会**被发送出去。

---

## ⚠️ 为什么不推荐录制模式发微信

项目的录制功能（`TaskRecorder`）记录的是**屏幕坐标和按键序列**，它存在以下致命缺陷：

1. **无法验证聊天对象**：录制时你点的是"张三"，回放时如果微信布局变化、分辨率变化，可能点到"李四"
2. **坐标漂移**：窗口大小、分辨率、DPI 缩放都会导致坐标偏移
3. **无法识别相似联系人**：搜索"张三"时，如果列表同时出现"张三"和"张三-工作群"，录制回放会固定点同一个坐标，但微信搜索结果顺序可能变化

**结论**：录制模式仅适用于无需身份验证的通用桌面操作（如打开记事本、点击计算器），**绝不推荐用于微信发送消息**。

---

## ✅ 唯一推荐方式：`WeChatSmartSender`

我们提供了专门为零误发设计的 `WeChatSmartSender`，它采用**视觉感知 + OCR 验证**的方式，而不是死记坐标。

### 安全等级

#### `STRICT`（默认，强烈推荐）

发送流程中的 5 重安全保障：

```
1. OCR 精确匹配搜索结果列表
   └─ 如果存在相似联系人冲突（如"张三" vs "张三-工作群"）→ 立即停止
2. 验证当前聊天窗口标题 == 目标联系人
3. 点击输入框并输入消息
4. 【发送前最终确认】再次 OCR 验证聊天对象（按 Enter 前的最后一道防线）
5. 按 Enter 发送后，验证消息气泡出现在右下角聊天区域
```

**任何一步失败 → 立即停止，不发送。**

#### `NORMAL`

比 `STRICT` 少两道防线：
- 允许 OCR 模糊匹配联系人
- 不检测相似联系人冲突
- 没有发送前的最终确认

适用于对速度有要求、且联系人名称非常独特的场景。

## 🔧 如何使用

### 命令行直接发送

```bash
# 严格模式（默认）
py wechat_send.py "文件传输助手" "测试消息"

# 标准模式
py wechat_send.py "文件传输助手" "测试消息" --safety-level normal
```

### 自然语言发送

```bash
py wechat_voice_assistant.py "给张三发消息说晚上好"
```

### 先用 Dry-Run 测试（重要！）

在第一次给某个联系人发送前，强烈建议先用 `--dry-run` 测试：

```bash
py wechat_send.py "文件传输助手" "测试消息" --dry-run
```

Dry-run 会执行到"发送前最终确认"这一步，验证所有 OCR 流程是否正常，但**不会真的按 Enter 发送**。如果 dry-run 成功，实际发送的成功率就非常高。

建议对新联系人连续 dry-run 3~5 次确认稳定后再实际发送。

## 🛡️ 安全机制详解

### 1. 相似联系人冲突检测

使用 `difflib.SequenceMatcher` 计算名称相似度，如果搜索列表中同时存在：
- 目标名称本身
- 与目标名称相似度 >= 80% 的其他结果
- 或者目标名称是其他结果的子串（如"张三" vs "张三-工作群"）

则 `STRICT` 模式会**拒绝自动选择**，报告冲突并停止。

**为什么这很重要？**
> 假设你要给"张三"发一条私密的借钱消息，但列表里同时有"张三-工作群"。如果没有冲突检测，自动点击可能点错，导致消息发到工作群，造成严重后果。

### 2. 聊天对象二次验证

点击联系人后，系统会截图聊天窗口的标题栏区域，通过 OCR 确认当前聊天对象的名字。如果 OCR 没有识别到目标联系人名称，立即停止。

这防止了以下情况：
- 点击联系人时没点中，聊天窗口没切换
- 微信卡顿，搜索结果列表还没更新

### 3. 发送前最终确认

在输入完消息、即将按 Enter 前，系统会**再次**截图并 OCR 验证聊天对象。这是最后一道防线，确保即使前面的步骤被某种意外绕过，发送前还有一次确认机会。

### 4. 消息发送后验证

发送后，系统会截图右侧聊天区域，通过 OCR 查找消息内容是否出现在右下角（自己发送的消息通常在右侧）。如果找不到，会报告"发送可能失败"。

## 📋 Troubleshooting

### "未精确匹配到联系人 'XXX'"

**原因**：
- 微信窗口没有聚焦
- 搜索框没有正确弹出
- 联系人名称输入有误（多字、少字、繁简体不一致）

**解决方法**：
1. 确保微信已登录且窗口可见
2. 手动按 `Ctrl+F` 搜索该联系人，确认能搜到
3. 检查名称是否完全一致（包括特殊符号、空格）
4. 尝试用 `--dry-run` 查看调试图（保存在 `logs/` 目录）

### "检测到相似联系人冲突"

**原因**：搜索列表中存在和你要找的联系人名字非常像的其他条目。

**解决方法**：
1. 修改目标联系人的微信备注名，使其唯一（如"张三"改成"张三-私聊"）
2. 或者使用 `--safety-level normal` 绕过冲突检测（安全性降低）
3. 手动发送

### "聊天对象验证失败"

**原因**：点击联系人后，聊天窗口没有正确切换。

**解决方法**：
1. 检查微信是否卡顿
2. 查看 `logs/wechat_validate_contact_fail_*.png` 调试图
3. 可能是某些微信版本的标题栏布局不同，尝试反馈问题

### "消息发送验证失败"

**原因**：消息可能已经发出，但 OCR 没有识别到；或者消息确实没发出去。

**解决方法**：
1. 查看聊天窗口确认消息是否真的发了
2. 查看 `logs/wechat_validate_msg_fail_*.png` 调试图
3. 如果消息确实发了但验证失败，可能是 OCR 对特殊字符/表情支持不好，属于误报

## 🚀 程序化调用

如果你要在自己的 Python 代码中调用：

```python
from src.utils.wechat_smart_sender import send_wechat_message_safe

# 严格模式
result = send_wechat_message_safe("文件传输助手", "Hello", safety_level="strict")
print(result.success, result.error)

# Dry-run 测试
result = send_wechat_message_safe("文件传输助手", "Hello", dry_run=True)
print(result.success, result.error)  # success=True, error="[DRY-RUN] 所有验证通过，未实际发送"
```

## 总结

| 方式 | 安全性 | 推荐度 | 适用场景 |
|:---|:---|:---|:---|
| `WeChatSmartSender` STRICT | ⭐⭐⭐⭐⭐ | ✅ 强烈推荐 | 微信发消息 |
| `WeChatSmartSender` NORMAL | ⭐⭐⭐⭐ | ⚠️ 可用 | 速度优先、名称唯一 |
| 录制回放 | ⭐⭐ | ❌ 不推荐 | 通用桌面操作 |
| VLM 自然语言 | ⭐⭐⭐ | ⚠️ 辅助 | 复杂指令解析 |

**请记住**：微信发消息宁可失败重试，也绝不能发错人。
